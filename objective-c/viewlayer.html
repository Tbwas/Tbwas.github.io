# 分析 UIView 的 frame

  • 概述
  • CALayer
  • UIView

# 概述

我们知道，基于 view 的绘制通常会调用drawRect:方法，该方法相对来说比较耗费性能，这是因为绘制操作是使用 CPU 在主线程上完成的。而基于 layer 的绘制是 Core Animation 在硬件上完成的，性能更优更高效。当然，App 中大多数的 layer 并不会做实际的绘制操作。相反，layer 通常捕获 App 提供的内容然后将其缓存在位图中。之后，当改变了 layer 其中一个属性时，所要做的只是改变与 layer 对象相关联的状态信息。

当一个改变触发一个动画时，Core Animation 会将 layer 的位图和状态信息传递给图形硬件，图形硬件根据最新的状态信息来渲染位图，在硬件中操作位图会产生比软件中更快的动画效果。在动画期间，Core Animation 会在硬件上逐帧绘制动画，而我们所要做的就是指定动画的开始点和结束点，剩下的全交给 Core Animation 来完成。

# CALayer

Layer 使用两种类型坐标系统：基于点的坐标系统和单元坐标系统。对于前者，我们会更熟悉一些，最常见的用途是指定了 layer 的位置和大小。即：

  • bounds 指定其在屏幕上的大小；
  • position 指定其相对父坐标系统中的位置；
  • frame 该属性是根据position、anchorPoint和bounds推导而来的；

而对于单元坐标系统，取值范围大小是 0.0 ~ 1.0，常见的属性是anchorPoint，默认值为 (0.5, 0.5)。

# UIView

当给 view 设置 frame 时，方法内部会调用-[CALayer setFrame:]和-[CALayer setPosition:]以及-[CALayer setBounds:]，而 position 是根据frame.origin和anchorPoint计算而来的。计算方式如下：

position.x = frame.origin.x + anchorPoint.x * frame.size.width;
position.y = frame.origin.y + anchorPoint.y * frame.size.height;

然而，在设置完 frame 后，如果我们只是改变了anchorPoint、view.center、layer.position、view.transform这些属性当中的一个或多个，再去获取 frame 时，你会发现 frame 竟然发生了变化！那么 frame 到底由什么决定的呢？

再弄清楚这个问题之前，先来看下 frame 到底是什么。根据苹果文档所述，frame 其实就是一个矩形框架，用来描述视图在父视图坐标系统中的位置和大小。重点是个矩形形状，所以对 view 应用旋转仿射变换后，frame 表示的仍然是包裹视图的矩形框架。通过断点调试可以看出，-[UIView frame]方法内部实现仅仅是调用了-[CALayer frame]：

也就是说，当获取 view 的 frame 时，其实是返回了 layer 的 frame。而 layer 的 frame 是由position、anchorPoint以及bounds这三者决定的。关系如下：

frame.size.width = bounds.size.width;
frame.size.height = bounds.size.height;
frame.origin.x = position.x - anchorPoint.x * frame.size.width;
frame.origin.y = position.y - anchorPoint.y * frame.size.height;

而 view 的center属性是跟 layer 的position属性相对应的，所以改变了view.center其实就是改变了layer.position。

当然，frame.size和bounds.size并不总是相等的。如果给 view 设置一个了 transform，那么两者便会发生差异。比如：

myView.frame = CGRectMake(50, 50, 100, 100);
myView.transform = CGAffineTransformMakeRotation(M_PI_4); // 顺时针旋转45°
1
2

此时，myView 的bounds依然是(0, 0, 100, 100)，但是frame却变成了(29.29, 29.29, 141.42, 141.42)。套用上述公式，可计算出frame。

frame.size.width = √(100^2 + 100^2) = 141.42;
frame.size.height = √(100^2 + 100^2) = 141.42;
frame.origin.x = 100 - 0.5 * 141.42 = 29.29; 
frame.origin.y = 100 - 0.5 * 141.42 = 29.29; // 锚点默认值为0.5
